§ 01 / これはなに

辞書の表紙と裏表紙を、
綴じてみた。A dictionary sewn into a ring.

ふつうの辞典は A→Z の線分。綴じた瞬間、Z のとなりが A になって輪になる。さらにひねれば表と裏が消えて、上位と下位、親と子の区別もなくなる。

わっかずかんは、その輪をそのまま UI にした分類・ナビゲーション装置。名前は三重掛詞 ——「輪っか」×「百科」×「輪化」。

既存の辞典もフォルダもタグも、迷子にならないかわりに旅ができない。
わっかずかんは、隣の輪に一歩踏み出すだけで意図しない場所に着く。
それでいて「今どのへんにいるか」は常に分かる。 — 設計原理
§ 01.5 / 二つの図鑑

ポケモンと、いきもの。Two datasets to start.

β版では2種類のデータセットを用意している。どちらも上部のタブで切り替えられる。同じUIで違う世界を旅すると、分類という行為そのものが目的ではなく、世界を理解する道具だということが体感できる。

§ 02 / 三つの発明

分類学は研究し尽くされている。
でも、ここは空白だった。Three novelties on top of Formal Concept Analysis.

デューイ十進分類法、ランガナタンのファセット分類、Wille の形式概念分析（FCA）—— 分類学は 100 年以上の蓄積がある。わっかずかんはその上に三点だけ新しいことを加える。

01 · SPLIT

分裂ルール

要素が追加されたとき、集合が自動で割れる。「わっかが われた！」という瞬間を子どもにも体感させる動的分類。

02 · LENS

レンズリング

一度に見せるのは「祖父母 → 自分 → 孫」の3世代だけ。認知科学の基底状態 = 3 に根ざしたビュー制約。

03 · MÖBIUS

メビウス接続

辞典の表紙と裏表紙を綴じて環化、さらにひねって表裏を消す。目的検索と漂流の境界がなくなる。

さわって、まわして、もぐる。Touch, turn, dive.

§ 04 / 分類と関係

木で分けた世界に、
グラフを一本通す。Trees for taxonomy. Graphs for kinship.

わっかずかんは基本的に木構造で分類する。くさ > そうげん > フシギダネ。ただし、世界は木だけでは表現できない。兄弟ポケモンは同じ一族でも、進化分岐すれば別タイプに散る。

この穴を埋めるのが円底の進化チェーンナビ。木の底辺に薄く一本、関係線が走る。線形家系は横並びの矢印で、イーブイ家のような分岐家系はファン状に開く。

どちらも単一データモデル gems[] で表現される。将来的な拡張（伝説 / バッジ / ステージ）は key を足すだけ。

イーブイ家（v45 で分岐対応）

カントー 151 匹のうち、唯一の 1→3 分岐。他 48 家系はすべて線形。

§ 05 / データ編集

CSV で遊ぶ。Export. Edit in Excel. Re-import. The tree rebuilds.

データを自分で差し替えられないツールはつまらない。v44 で 📥 Export / 📤 Import を実装した。1 プレパラート = 1 ファイルのパス方式。BOM 付き UTF-8 なので Excel でそのまま開ける。

フォーマット

id,category,name,emoji,desc,gems_extended
p001,くさ,フシギダネ,🌱,"くさタイプ・そうげんに...",
p004,ほのお,ヒトカゲ,🦎,"ほのおタイプ・やまに...",
p025,でんき,ピカチュウ,⚡,"でんきタイプ・もりに...",
...

フィールド

id
一意のレコードID既存の p001〜p151 と一致させれば上書き。新しい id は無視される
category
そのプレパラートでの分類バケットタイプなら「くさ」、すみかなら「そうげん」など。未知のカテゴリも ❓ 付きで自動吸収
gems_extended
追加のサブタグセミコロン区切りで key:value または key:value@order 形式。dex と evo_family は自動導出なので不要

注意: トレーナー図鑑はジムリーダー定義を触るので v1 では import 非対応。TRAINERS 定義を直接編集する。

§ 06 / 設計哲学

Boolean を Float にする。From 0/1 to a continuum.

わっかずかんの背後にあるのは、osakenpiro のすべての仕事を貫く一本の軸。「所属」か「非所属」かの二値判断を、どれくらい近いかの連続値に置き換える。

兄弟ズームは「隣はどれくらい似ているか」を体で学ばせる装置。ピン留め比較は、集合 A と集合 B の重なり具合を光で見せる。タイプ軸の「くさ」とすみか軸の「もり」は、論理的には別レイヤーだが、実際には同じポケモンを違う角度から眺めているだけ —— それが光の交差で見える。

分類は二値ではない。
個別最適の規格化、規格の個別最適化。 — osakenpiro 設計原則

関連資料

この UI の認知科学的根拠は、osakenpiro の論文シリーズ第 8 本「The Ground State of Three」で扱っている。FCA の拡張としての位置づけ（分裂ルール・レンズリング・メビウス接続の三点新規性）は、将来の情報科学系論文で展開予定。
